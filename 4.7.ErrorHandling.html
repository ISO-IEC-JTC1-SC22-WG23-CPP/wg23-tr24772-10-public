4.7 Error Handling

In C++, there are four mechanisms for error handling, exceptions, std::expected, std::error_code, and errno.

Exceptions are a mechanism to gather error handling mechanism for a given error in one place. If an error detected, an exception is raised and control transferred to a matching catch handler where an appropriate action can handled or repropagated. If no matching handler is found, std::terminate is called and the program terminates. A search for a matching handler stops at main(), a function declared noexcept, and the top level function passed to threading APIâ€™s. See ISO/IEC 14882 for the complete list.

In the case of failed assertions, calls to abort(), std::terminate, exit() similar functions, the program terminates without calling any destructors. For coroutines, a rethrown exception or an uncaught exception causes the coroutine to end and cannot be resume. The exception itself is not propagated.
